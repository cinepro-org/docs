Hey, nice to see you here! This section is for developers who want to contribute to the project.

## How to Contribute

CinePro is an open-source project, and contributions are always welcome! Here are some ways you can contribute:


<Card title="Report Bugs" icon="bug" href="#bug-reporting">
    If you encounter any issues, please report them on the <Link href="https://github.com/cinepro-org/backend/issues">GitHub Issues page</Link>.
</Card>

<Card title="Suggest Features" icon="lightbulb" href="#feature-requests">
    Have an idea for a new feature? Share it with us by creating a feature request on GitHub.
</Card>

<Card title="Submit Pull Requests" icon="code-branch" href="#pull-request-guidelines">
    If you have a fix or improvement, feel free to fork the repository, make your changes, and submit a pull request.
</Card>

### Bug Reporting

If you encounter an error or bug, please follow these steps to help us resolve the issue quickly:

<Steps>
    <Step title="Update to the newest version" icon="cloud-arrow-down">
        Check if you are using the latest version of the API. If not, update to the latest version.
    </Step>

    <Step title="Check if the issue is already reported" icon="check">
        Check if the issue is already reported on the [GitHub Issues page](https://github.com/cinepro-org/backend/issues) or if it has been mentioned somewhere in the [discussions](https://github.com/orgs/cinepro-org/discussions). 
    </Step>

    <Step title="Create a new issue" icon="bug">
        <Warning>
            ⚠️ Please do not create a new issue if the issue is already reported. ⚠️ 
        </Warning>
        If the issue is not reported, please create a new issue on the [GitHub Issues page](https://github.com/cinepro-org/backend/issues/new) with a detailed description of the error and the steps to reproduce it (also possible cause and if you are connected to a VPN or simmilar).
    </Step>
</Steps>

### Feature Requests

If you have an idea for a new feature or improvement, please check if it has already been suggested. If not, feel free to create a new issue on the [GitHub Issues page](https://github.com/cinepro-org/backend/issues/new) with a detailed description of the feature and its potential benefits.

### Pull Request Guidelines

When submitting a pull request, please follow [CinePro's coding standards](https://github.com/cinepro-org/backend/blob/main/.github/CONTRIBUTING.md).

Please create an issue first to discuss the changes you want to make. This helps us understand your intentions and ensures that your changes align with the project's goals.
